Todo List Endpoint

This is a sample Concordion specification page demonstrating features of the Concordion REST extension.
To-Do List Endpoint is a rich RESTful API that enables you to manage your day-to-day tasks.
Currently, following operations are supported:
  • Create To-Do item
  • Fetch particular To-Do item
  • Complete To-Do item
  • Delete To-Do item
  • Fetch entire To-Do list
  • Export To-Do list in PDF format
  • Export To-Do list in XML format

Examples:

Creating To-Do item
When I send /tasks with following body (using application/json) 
{
  "description": "Buy the milk",
  "priority": 3
}
Then I should receive HTTP/1.1 201 Created with response header /tasks/1
Fetchnig To-Do item
Warning: This example was intentionally left failing in order to demonstrate JSON comparition view feature.
When I successfully send /tasks with following body (using application/json) 
{
  "description": "Walk the dog"
}
Then /tasks/1 should return HTTP/1.1 200 OK with response containing: 
{
  "task_id": 1,
  "completed": false,
  "description": "Walk the dog",
  "priority": 1
}
{
  "task_id": 1,
  "completed": false,
  "description": "Walk the dog",
  "priority": 0
}
Completing To-Do item
Given there is pending To-Do item with id 1
When I send /tasks/1/complete
Then I should receive HTTP/1.1 200 OK
And /tasks/1 should return HTTP/1.1 200 OK with response containing: 
{
  "task_id": 1,
  "completed": true
}
Deleting To-Do item
Warning: This example was intentionally left failing in order to demonstrate HTTP status comparition feature.
Given there is To-Do item with id 1
When I send /tasks/1
Then I should receive HTTP/1.1 200 OK HTTP/1.1 204 No Content
And /tasks/1 should return HTTP/1.1 404 Not Found
Given there are following tasks:
TaskID Description Priority Completed
1 Buy the milk 1 Y
2 Walk the dog 3 N
3 Visit the gym 2 N
Listing To-Do items as JSON (default)
When I send /tasks
Then I should receive HTTP/1.1 200 OK with response containing: 
{
  "tasks": [
    {
      "task_id": 1,
      "description": "Buy the milk",
      "priority": 1,
      "completed": true
    },
    {
      "task_id": 2,
      "description": "Walk the dog",
      "priority": 3,
      "completed": false
    },
    {
      "task_id": 3,
      "description": "Visit the gym",
      "priority": 2,
      "completed": false
    }
  ]
}
Listing To-Do items as PDF
When I send /tasks with application/pdf
Then I should receive todo-list.pdf with application/pdf and header attachment; filename=todo-list.pdf
Listing To-Do items as XML
Warning: This example was intentionally left failing in order to demonstrate HTTP header comparition feature.
When I send /tasks with application/xml
Then I should receive HTTP/1.1 200 OK with application/xml (not set) and response body containing:

    <todo-item task-id="1" completed="true">
